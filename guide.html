מה בנינו

CourtIQ היא פלטפורמת אימון כדורסל מבוססת בינה מלאכותית. המטרה: לתת לשחקנים בכל הרמות תוכנית אימון אישית, מעקב אחר התקדמות, וניתוח ביצועים — הכל דרך AI.

דף נחיתה שיווקי
index.html — מוכר את המוצר, כולל תמחור, תכונות, ו-CTA
דשבורד ראשי
app.html — האפליקציה עצמה, 10 פאנלים שונים
Backend ב-Supabase
מסד נתונים, אימות משתמשים, Edge Function
AI Coach
Claude API מחובר דרך Edge Function מאובטחת

שלבי הפיתוח לפי Git

שלב 1 — Initial release · CourtIQ גרסה ראשונה
שלב 2 — UX redesign · פיצול ל-index/app, עיצוב מלא, תמחור
שלב 3 — Supabase integration · DB, Auth, Edge Function
שלב 4 — SQL helpers · כלי עזר להקמת מסד הנתונים

הסטאק הטכנולוגי

Frontend — צד הלקוח

HTML5
שפת המבנה של כל דף ווב. כל האפליקציה בשני קבצי HTML גדולים — index.html ו-app.html. בחרנו Vanilla JS ללא React/Vue לפתיחה מהירה ללא כלי build.
CSS3
CSS Variables לניהול צבעים, Flexbox ו-Grid לפריסה, Animations לחוויה חלקה. Breakpoints: Desktop 1024px+ / Tablet 768px / Mobile <768px.
JavaScript (Vanilla)
State management דרך Object גלובלי APP, Event listeners, Async/Await לקריאות API, DOM Manipulation לשינוי תצוגה.
Chart.js
ספריית גרפים מוכנה. 4 גרפים בפאנל Analytics: קו (אחוז זריקה), עמודות (השוואה שבועית), קו (קפיצה אנכית), קו (ספרינט).

Backend — צד השרת

Supabase — Backend-as-a-Service שנותן מסד נתונים, אימות, ו-API תוך דקות, ללא צורך לכתוב שרת. בחרנו בו על פני Firebase בגלל PostgreSQL אמיתי, Row Level Security מובנה, ו-Edge Functions מבוסס Deno.
שירותמה זהמה עשינו
Supabase Authניהול משתמשיםהרשמה, כניסה, session
Supabase DatabasePostgreSQL בענןשמירת פרופילים, אימונים, שבועות
Supabase Edge Functionsשרת קוד בענןProxy ל-Claude API

Row Level Security (RLS)

מדיניות אבטחה ישירות במסד הנתונים. בלי RLS, אם יש bug בקוד, משתמש יכול לראות נתוני משתמשים אחרים.

-- המדיניות שכתבנו:
create policy "sessions: select own"
  on training_sessions
  for select
  using (auth.uid() = user_id);
-- = "משתמש יכול לראות רק שורות שה-user_id שלהן שווה ל-ID שלו"

מסד הנתונים

טבלה 1: profiles

מי הוא המשתמש.

profiles
├── id           (UUID)     ← מזהה ייחודי = זהה למזהה ב-auth.users
├── first_name   (text)     ← שם פרטי
├── last_name    (text)     ← שם משפחה
├── position     (text)     ← PG / SG / SF / PF / C
├── skill_level  (text)     ← Beginner / Intermediate / Advanced
├── goal         (text)     ← מטרת האימון
├── plan         (text)     ← starter / pro / elite
├── streak       (int)      ← כמה ימים ברצף אימן
├── created_at   (timestamp)
└── updated_at   (timestamp)

טבלה 2: training_weeks

שבוע אימון שהסתיים וקיבל סיכום AI.

training_weeks
├── id           (UUID)     ← מזהה ייחודי לשבוע
├── user_id      (UUID)     ← מפנה ל-auth.users (Foreign Key)
├── week_number  (int)      ← מספר השבוע (1, 2, 3...)
├── label        (text)     ← תווית מוצגת "W4"
├── summary_json (jsonb)    ← כל ניתוח ה-AI שמור כ-JSON
└── created_at   (timestamp)

מה summary_json מכיל:

{
  "averages": {
    "shootingPct": 67.3,
    "verticalIn":  28.5,
    "sprintSec":   4.2
  },
  "strengths":   ["Shooting consistency", "Sprint speed"],
  "focusAreas":  ["Vertical jump needs work"],
  "feedback":    "טקסט AI מפורט על השבוע...",
  "generatedAt": "2025-03-01T10:00:00Z"
}

טבלה 3: training_sessions

אימון יומי בודד.

training_sessions
├── id               (UUID)
├── user_id          (UUID)    ← Foreign Key ל-auth.users
├── week_id          (UUID)    ← Foreign Key ל-training_weeks (אופציונלי)
├── day              (text)    ← "Mon", "Tue", "Wed"...
├── shots_made       (numeric) ← זריקות שנכנסו
├── shots_attempted  (numeric) ← זריקות שנזרקו
├── dribbling_min    (numeric) ← דקות דריבלינג
├── vertical_in      (numeric) ← קפיצה אנכית באינצ'ים
├── sprint_sec       (numeric) ← זמן ספרינט בשניות
├── intensity        (text)    ← Low / Medium / High
├── notes            (text)    ← הערות חופשיות
└── created_at       (timestamp)

Foreign Keys ו-Trigger

Foreign Key — CASCADE
training_sessions.user_id → auth.users.id

אם מוחקים משתמש → כל הסשנים שלו נמחקים אוטומטית.
Trigger — יצירת פרופיל
כשמשתמש נרשם ל-auth.users, Trigger יוצר שורה ב-profiles אוטומטית — גשר בין טבלאות Supabase הפנימיות לטבלאות שלנו.

אינטגרציית ה-AI

למה צריך Edge Function?

בעיה: Claude API Key חייב להיות סודי. אם נשים אותו ב-HTML — כל אחד יכול לגנוב אותו.

פתרון: הקוד שרץ בשרת (Edge Function) שומר את ה-key סודי. הפרונטאנד שולח בקשה לEdge Function → זו שולחת לClaude.

זרימת יצירת הסיכום השבועי

שלב 1
איסוף סשנים מ-Supabase DB עבור שבוע זה
שלב 2
בניית System Prompt + User Message עם הנתונים
שלב 3
קריאה ל-Edge Function עם Authorization header
שלב 4
Edge Function שולחת ל-Claude API (API Key נסתר)
שלב 5
שמירת summary_json בטבלת training_weeks

קוד הקריאה לEdge Function

const response = await fetch(
  'https://[project].supabase.co/functions/v1/claude-proxy',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${session.access_token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ messages, system: systemPrompt })
  }
)

System vs User Messages

System Message
הוראות קבועות לAI — מי הוא, מה התפקיד שלו, הקשר המשתמש (פוזיציה, רמה).
User Message
הבקשה הספציפית — הנתונים של השבוע שהוא צריך לנתח.

מה חסר

A. בעיות אבטחה

בעיהחומרההסבר
אין auth guard ב-app.htmlקריטיכל אחד יכול להיכנס ללא התחברות
innerHTML ללא sanitizationגבוההסיכון XSS — הזרקת קוד זדוני
Supabase anon key נגלהבינוניתמקובל בחלקו אבל חשוב להיות מודע
XSS מוסבר: תוקף שולח הודעת צ'אט: <script>sendAllData(hackerSite)</script>. אם משתמשים ב-innerHTML ישירות — הסקריפט מופעל. פתרון: textContent במקום innerHTML, או ספריית DOMPurify.

B. תכונות שמופיעות אבל לא פועלות

תכונהמה קייםמה חסר
Video Drillsממשק יפהוידאו אמיתי, storage
Game Film AnalysisUpload UIbackend, AI analysis
Nutrition & Recoverymock dataAI generation, persistence
CalendarUI בסיסילוגיקת לוח שנה, scheduling
Notificationsפאנל ריקכל המערכת

C. בעיות טכניות

אין error handling
המשתמש לא יודע כשמשהו נכשל
Chat history ב-memory בלבד
נאבד ברענון הדף
אין validation בטפסים
ניתן לשלוח נתונים ריקים לDB
הכל בקובץ HTML אחד
קשה לתחזוקה ועדכונים

מפת דרכים לאפליקציה המושלמת

1
Phase 1 — יסודות
2–3 שבועות · לוודא שמה שקיים עובד כמו שצריך
Auth Guard — בדיקת session בכניסה ל-app.html, redirect לLogin אם לא מחובר
Error Handling — toast notifications לכל שגיאה, console.error לדיבאג
Chat History Persistence — טבלת chat_messages בDB, שמירת כל שיחה
Input Sanitization — DOMPurify לכל innerHTML, validation בטפסים
2
Phase 2 — תכונות ליבה
3–5 שבועות · להגיש מה שמובטח ב-PRO ו-ELITE
Video Drill Library — Supabase Storage, טבלת drills, HTML5 video player
Game Film Analysis — Upload לStorage, ניתוח עם Claude, דוח PDF
Nutrition & Recovery — שאלון, Edge Function ליצירת תפריט שבועי עם AI
Calendar עם AI — ספריית Flatpickr, lוגיקת scheduling, המלצות AI
3
Phase 3 — פולישינג
2–3 שבועות · חוויית משתמש מלאה ומלוטשת
Push Notifications — Web Push API + Service Worker, תזכורות אימון
Data Export — CSV ו-PDF מכל ה-sessions, גרפים להורדה
PWA — manifest.json + service-worker.js, "התקנה" כאפליקציה בטלפון
Accessibility — ARIA labels, contrast ratio, ניווט מקלדת
4
Phase 4 — Scale
חודש+ · להפוך לעסק אמיתי
Payment — Stripe — Checkout, Webhooks, עדכון plan אוטומטי בDB
Admin Panel — ניהול משתמשים, תוכן, analytics ו-revenue
Social Features — Leaderboard, Training Challenges, Coach Assignment
Native Mobile — Capacitor (מומלץ) עוטף את האתר הנוכחי כApp
5
Phase 5 — AI Upgrade
שנה+ · AI ברמה הבאה
Pose Estimation — Google MediaPipe, ניתוח פורמת יריה דרך מצלמה
Personalized AI — Prompt דינמי בנוי על היסטוריה אישית, זיהוי דפוסים
Streaming Responses — הצגת תשובת AI מילה-מילה במקום להמתין לכל התשובה
